READ ME

Per il progetto di LTW, il nostro gruppo ha creato una piattaforma di trading di criptovalute con moneta proprietaria - i SapienzaCoin. Ci siamo divertiti molto a svilupparla perché abbiamo imparato da zero moltissime tecnologie e integrarle tutte in un unico prodotto è stato davvero soddisfacente.

Abbiamo strutturato il sito come un "GitHub dinamico", quindi:
 -  al paragrafo Technologies  trovate una descrizione dettagliata delle tecnologie usate con link ad esempi;
 -  al bottone GitHub  trovate la repo con il codice;
 -  al bottone About us  trovate informazioni su noi sviluppatori;


TECHNOLOGIES  ≈  3k locs

 1. Front-end:

   HTML   +  CSS   ==  1168 locs per HTML + 628 locs per CSS

Abbiamo usato HTML per la struttura della pagina web e CSS per la personalizzazione dei contenuti. In totale abbiamo creato 3 pagine principali: homepage, portfolio e about_us. Ogni pagina è organizzata con la propria coppia di file HTML e CSS e per strutture particolarmente complicate come il grafico di portfolio, abbiamo creato appositamente un coppia HTML - CSS per gestire quel particolare elemento.

    JAVASCRIPT   +  JQUERY   ==  569 locs

La maggior parte del codice JavaScript è stato scritto per il grafico e per le tabelle nella pagina di portfolio. Ognuno di loro ha associato un file Javascript che si occupa dell'eleborazione dei dati del DB o delle crypto e della costruzione delle tabelle e del grafico in base ai dati estrapolati. In aggiunta a questo, alla fine del body delle pagine HTML ci sono degli script JQuery, Ajax e JavaScript che gestiscono eventi e animazioni dinamiche.

   BOOTSTRAP

Escludendo le tabelle e il grafico di portfolio, tutti gli oggetti nel nostro sito sono oggetti Bootstrap. Questo garantisce un sito responsive e un'alta retrocombatibilità con CSS, JS e JQ, che abbiamo sfruttato per personalizzare la grafica dell'elemento Bootstrap in accordo con lo stile del sito. Alcuni elementi utilizzati sono puramente statici - come i flex, le grid e i container che incapsulano il sito - mentre altri possiedono anche una parte dinamica - come il carosello all'inizio della homepage e i vari modal nella homepage e in portfolio.

 2. Back-end:

   FLASK   +  PYTHON   ==  432 locs

Python è il linguaggio di scripting back-end che abbiamo deciso di utilizzare per il server e per interagire con il database. Il server consiste in un file Python che utilizza la libreria Flask per interagire con il front-end. In questo file viene coordinata tutta la logica del sito, ci sono le richieste e modifiche al database e le rotte verso le varie pagine. Le rotte sono adornate da un decoratore che controlla l'autenticazione dell'utente e la correttezza della sua Apikey. Se il controllo va a buon fine allora l'utente può entrare nel proprio portfolio, altrimenti usiamo un Flash di errore e rindirizziamo all'homepage. Per garantire la sicurezza dei nostri utenti, abbiamo implementato il protocollo SALT in cui generiamo uno SHA256 basato sulla cifratura dei cookie dell'utente.

   SQLALCHEMY

SQLAlchemy è un DB toolkit per Python, che usa un database relazionale SQL. Abbiamo 3 tabelle: User, Storico e Moneta. Ogni sign up popola il DB con un nuovo User e a questo vengono associati tramite foreign key una tabella Storico con lo storico delle proprie transazioni e una tabella globale Moneta con i valori di mercato delle crypto. Supportiamo le operazioni di buy e sell che possono essere chiamate dai modal in nella pagina di portfolio e modificano le tabelle Storico e il wallet dell'utente.

    D3JS  TECHAN  COINMARKETCAP

Le tabelle e il grafico in portfolio sono basati su dati reali, presi da CoinMarketCap - una piattaforma di scambio di criptovalute. Techan è una libreria di algoritmi finanziari che elebaora i dati del mercato e restituisce i dati per formare il candlestick, l'OHLC, l'ATR e il volume graph che vengono poi plottati da D3JS e forniti di funzioni di zooming e modifica live.
